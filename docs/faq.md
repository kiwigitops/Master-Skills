# FAQ

## Is this an awesome list?

No. This project is schema-first. Markdown indexes are generated from structured YAML entries.

## Can I add non-GitHub tools?

Not yet. Current schema requires `github_url` for consistency.

## How do I decide a category?

Use the project's primary job, not marketing language. See `docs/classification-system.md`.

## What if a project fits multiple categories?

Pick one canonical `category` and use `tags`, `use_cases`, and `related_projects` to capture overlap.

## Why include limitations?

Limitations prevent one-sided entries and help users make realistic choices.

## What does beginner-friendly mean?

A project is beginner-friendly only if setup, docs, examples, and troubleshooting are clearly available.

## Are the examples production recommendations?

No. Files under `examples/` are explicitly labeled examples.

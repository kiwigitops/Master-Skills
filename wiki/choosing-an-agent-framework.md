# Choosing An Agent Framework

## Step 1: Define your primary outcome

Choose one first:
- build software faster
- automate research and synthesis
- automate browser or desktop tasks
- orchestrate enterprise workflows

## Step 2: Match complexity

- low complexity: single-agent + simple tool calls
- medium complexity: retries, memory, and structured outputs
- high complexity: role-based multi-agent orchestration

## Step 3: Compare practical criteria

Use these fields in the data entries:
- `maturity`
- `activity_status`
- `docs_quality`
- `beginner_friendly`
- `limitations`

## Step 4: Run a pilot

Use one real task, not a toy prompt. Measure:
- success rate
- setup effort
- runtime cost
- failure recovery

## Step 5: Decide adoption

Adopt only after:
- repeatable success on your own task
- acceptable operational overhead
- clear path for contributors

# Beginners Guide

This guide is for people who are new to AI agents.

## What is an AI agent?

An AI agent is software that can do tasks for you, not just answer questions.

A chatbot usually responds once.
An agent can also:
- decide next steps
- call tools
- use memory
- keep working until a task is done

## Agent vs framework vs skill

- Agent: the worker that does tasks.
- Framework: the toolbox used to build and run agents.
- Skill: a reusable instruction package that helps agents perform a specific job better.

## How to choose tools

Start with your use case:
1. What do you want automated?
2. Do you need tool calls (APIs, browser, files)?
3. Do you need memory between sessions?
4. Do you need team features like logs and permissions?

Then compare options using:
- category
- maturity
- limitations
- beginner-friendly flag

## Common mistakes

- picking the most popular tool before defining the problem
- ignoring limitations and edge cases
- using advanced multi-agent setups too early
- skipping evaluation and observability

## A simple starter path

1. Pick one coding agent or framework.
2. Run one small workflow end-to-end.
3. Add validation and logging.
4. Add memory only when needed.
5. Expand to multi-agent only after single-agent stability.
